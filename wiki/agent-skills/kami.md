# Kami

A document design system packaged as an agent skill. Six templates cover one-pager, resume, slides, portfolio, letter, and long-form doc. Works across Claude Code, Codex, and Claude Desktop — invoked in natural language, no slash commands.

- **Repo:** [tw93/Kami](https://github.com/tw93/Kami)
- **Stars:** ~1.9K (2026-04-22, 2 days after launch)
- **Created:** 2026-04-20
- **Language:** HTML

## Why It Matters
- Demonstrates that a single skill can span multiple harnesses (Claude Code, Codex, Claude Desktop) — evidence the skill format is becoming portable infra, not a Claude-Code-only construct.
- "Invoked via natural language, no slash commands" — a deliberate UX choice: discoverability by intent rather than by memorised command.
- Template-per-document-type is an under-explored skill shape compared with tool-oriented skills ([[caveman]], [[darwin-skill]]).

## Key Takeaways
- Cross-harness skills are viable when the packaging stays file-based and runtime-agnostic.
- Template bundles (six doc types in one) scale better than one-skill-per-format for users.
- Sits adjacent to [[html-ppt-skill]] (slides) and the Claude Code design cluster ([[huashu-design]], [[diagram-design]]).
