# diagram-design

Thirteen editorial-quality diagram types for Claude Code. Self-contained HTML + SVG output. Explicit positioning: "No shadows, no Mermaid-slop."

- **Repo:** [cathrynlavery/diagram-design](https://github.com/cathrynlavery/diagram-design)
- **Stars:** ~1.4K (2026-04-22)
- **Created:** 2026-04-16
- **Language:** HTML

## Why It Matters
- Direct critique of Mermaid as a rendering target — the skill is positioned as a *taste* upgrade over the default agent-generated diagrams.
- Sits next to [[fireworks-tech-graph]] as the second diagram-focused Claude Code skill; together they suggest diagrams are becoming their own skill category rather than a feature of doc skills.
- Part of the 2026-04-22 Claude Code design cluster (with [[huashu-design]], [[awesome-claude-design]], [[open-codesign]]).

## Key Takeaways
- "Editorial" is the recurring keyword — presentation quality is now a discriminator between skills.
- HTML + SVG as deliverable format (no Mermaid, no image generation) is a shared stance across the 2026-04 design cluster.
- Thirteen fixed diagram types beats unbounded generation for consistency — a curated vocabulary trade-off.
